• Hassan Soliman, Heike Adel, Mohamed H. Gad‐Elrab, Dragan Milchevski, Jannik Strötgen. ”A Study on Entity Linking Across Domains: Which Data is Best for Fine‐Tuning?”. Proceedings of the 7th Workshop on Representation Learning for NLP (RepL4NLP’22). Accepted Blind Submission.


Abstract

Entity linking disambiguates mentions by mapping them to entities in a knowledge graph (KG). One important question in today's research is how to extend neural entity linking systems to new domains. In this paper, we aim at a system that enable linking mentions to entities from a general-domain KG and a domain-specific KG at the same time. In particular, we represent the entities of different KGs in a joint vector space and address the questions of which data is best suited for creating and fine-tuning that space, and whether fine-tuning harms performance on the general domain. We find that a combination of data from both the general and the special domain is most helpful. The first is especially necessary for avoiding performance loss on the general domain. While additional supervision on entities that appear in both KGs performs best in an intrinsic evaluation of the vector space, it has less impact on the downstream task of entity linking.


• This work is based on my Master's thesis. More details can be found below in its brief explanation.



Cross-Domain Neural Entity Linking In brief





The high-level topic of my Master’s thesis is "Cross-domain Neural Entity Linking" (CDNEL). The focus is on domain-agnostic or easy-to-extend-domains approaches using contextual-aware embeddings by BERT.


In the thesis work, the situation is that there exists domain-specific documents with extracted mentions, which need to be linked to entities from a general-domain KG (Wikipedia) and domain-specific KG simultaneously, as shown in the figure.


The task is to use state-of-the-art neural entity linking models, which are robust in terms of mean average precision (MAP) when evaluated on mentions that should be linked to general-domain KG, and extend these models to include a domain-specific KG, while keeping the evaluation levels the same on mentions that should be linked to this domain-specific KG.


The action I took is to do literature review for these models, and there exists a state-of-the-art neural entity linking model that suits the problem. It assumes the input mention is surrounded with textual context, and the entities in any given KG have textual descriptions.





In the candidate generation phase, it uses BERT in a Biencoder (two twin encoders) architecture, one encoder is used to encode the input mention with textual context, the other encoder is used to encode each of the entities in a KG with their textual descriptions. Using semantic similarity between the user input encoding and each of the entities encodings. Retrieval of the top similar 10 candidates is done.





In the candidate ranking phase, it uses BERT in a Crossencoder architecture which takes an input mention with textual context concatenated with each of the top 10 similar candidates with descriptions. It tries to learn the joint-context between them. Afterwards, each of the candidate entity has a similarity score, they are ranked from the most relevant to the least.


My contribution is to experiment which data is best for fine-tuning the model on a new domain-specific KG to be added and extended to the model. In addition, the overlapping entities, e.g., Clara Oswald shown in the first figure, are extracted and used in the fine-tuning to have similar representations. These experiments result in 4 different variants (T, TO, TS, TOS) of CDNEL. These variants are evaluated on separate documents that should be linked to entities from the new domain-specific KG and the original general-domain KG.


The results are promising as shown in the below figure. I did the experiment against four domain-specific KGs. There is an improvement in MAP compared with the baseline model (BLINK). The fine-tuned models do not overfit to the domain-specific KGs (target KGs) as well, since their MAP is still the same or slightly better compared with the baseline when evaluated on the general-domain KG (source KGs).

Egyptian Car License Plate Information Detection

This project is concerned with cars exceeding speed limit captured by the radar in Egypt, main idea is to automate this system by implementing an application which extract license information from these images. Our point of concern is to apply the process on Arabic license plates. We prepared a dataset for various kinds of car plates found in Egypt, and we tried to cover all possible real-life conditions. After that we made a survey to know any kind of work related to ours, then we prepared a structure and a full documentation for it. Finally, we started the implementation phase in which we were divided into three groups working in three different modules. First one is Object Detection for Vehicles in an image, then Classification + Localization for license plates, then Semantic Segmentation for the letters and numbers, finally Image Recognition for these characters. Me together with my team approached these problems by using a combination of classical Machine Learning algorithms at first as an initial step, then migrated to Deep Learning with a heavy weight of using pretrained CNN models to do Transfer Learning and fine-tune them to suit the project purposes.
